블로그 작성의 핵심 공리 (Core Blog Writing Principles)

=== 제1공리: 키워드 의도 불일치 = 즉시 사망 ===
• 알고리즘은 사용자 문제 해결사로 작동
• 검색 의도와 콘텐츠 주제가 어긋나면 양쪽 모두에게 버림받음
• "구미흥신소" 검색자 = 해결사를 찾는 상업적 의도
• "대처법" 콘텐츠 = 정보성 의도
• → 둘의 불일치 = 알고리즘 신뢰 파괴

=== 제2공리: 키워드 구현 - 반복에서 맥락으로 ===
• 알고리즘은 키워드를 세는 계수기가 아님
• 현대 알고리즘은 맥락(Context)과 자연스러움 평가
• 1,500자 기준 핵심 키워드 3-5회(제목포함) 자연스럽게
• 세는 것을 멈춰라. 인위적 반복 = 스팸 처리
• 형태소 분석: '포렌식', '포렌식을', '포렌식은' = 동일 인식
• 의미론적 확장이 단순 반복보다 강력함

=== 제3공리: 2분할 포스팅 = 구조적 자살 ===
• 스마트블록은 완결된 정보를 요구
• 분할된 글 = 불완전한 문서로 인식
• 체류시간 짧아짐 (각 편 30초-1분)
• 강제 링크 이동 = 사용자 경험 파괴
• 키워드 잠식(Cannibalization) 효과 발생
• 무조건 단일 완결형 '파워 페이지' 구축

=== 제4공리: 제목 최적화 - 모바일 퍼스트 ===
• 모바일 1줄 노출: 13-15자 (안전구간)
• 모바일 2줄 한계: 22-26자 (말줄임 회피)
• F-존 법칙: 최초 10자가 승부처
• 조사 생략으로 키워드 밀도 최적화
• "강남역에 있는 맛집들" → "강남역 맛집 추천"

=== 절대 법칙: 1포스트 1의도 원칙 ===
• 하나의 게시물 = 단 하나의 핵심 의도만 목표
• 상업적 키워드면 → 전문 서비스 소개서 형태
• 정보성 키워드면 → 순수 정보 제공
• 혼재 시도 = 즉시 패널티

=== 지역 키워드 필수 구조 ===
• 제목: [청주 흥신소] 핵심 키워드 최전방 배치
• 본문: 서두+결론에 지역명 1-2회 자연 언급  
• 지도: 네이버 스마트플레이스 등록된 사업장 지도 필수
• 태그: #청주흥신소 #청주민간조사
• → 일관된 지역 신호가 알고리즘 신뢰 구축

=== 치명적 금지사항 ===
• 제목→본문 서두 그대로 복붙 = 10년전 스팸 기법, 즉시 자폭
• 키워드 제목 2회 반복 = 키워드 스터핑, 페널티 확정
• 과도한 감정적 어조 = "심장이 내려앉고" 같은 표현 금지
• 무관한 지도 삽입 = "구미 경찰서" 같은 엉뚱한 지도
• 2분할/시리즈 연재 = 체류시간 파괴, 이탈률 급증
• 의미적 포화 = LSI 키워드 억지로 쑤셔넣기
• 냉철한 분석가 톤 = 공감 부재로 신뢰 형성 실패
• 발행 후 글 수정 = 신뢰도 리셋, 랭킹 초기화 (제5공리)
• 외부 급증 트래픽 = 이상 트래픽으로 오인, 보호조치 (제6공리)
• 품앗이 소통 = 인위적 네트워크로 C-Rank 하락
• 텍스트 세탁 생략 = RTF 태그 오염으로 스팸 분류 (제7공리)
• 동일 CTA 배너 반복 = 해시 중복으로 저품질 직행 (제8공리)

=== 제5공리: 글 수정 = 신뢰도 리셋 (절대 금지) ===
• 발행 후 수정 = 베이트 앤 스위치 필터 작동
• 제목 수정 = 치명적, 즉각적인 순위 하락
• 외부 링크 추가 = 스팸 분류, 쉐도우 밴 위험
• 이미지 교체 = 해시값 변경으로 이미지 검색 유입 중단
• 골든타임: 발행 후 10-30분 내 수정만 허용
• 수정할 바에는 비공개 후 새로 발행

=== 제6공리: 외부 유입 트랩 회피 ===
• 카톡/인스타 급증 트래픽 = 이상 트래픽 탐지
• 3-5초 이탈률 높음 = D.I.A. 점수 하락
• 스낙 컬처 사용자들의 낮은 체류시간 문제
• 외부 홍보할수록 네이버 내부 랭킹 하락 역설
• 검색 유입 비중을 탄탄히 다져놓는 것만이 완충재

=== 제7공리: 텍스트 세탁 (필수 위생절차) ===
• 외부 텍스트 = RTF 태그, 제로폭 문자 오염
• 워드/ChatGPT 복사 = 숨겨진 HTML 코드 포함
• Mac: 텍스트편집기 → 일반텍스트 만들기 (⇧⌘T)
• Windows: 메모장 경유하여 모든 서식 제거
• U+200B 제로폭 문자 제거 필수
• [외부 작성 → 플레인텍스트 변환 → 특수문자 검수 → 네이버 에디터]

=== 제8공리: 이미지 해시 세탁 ===
• 동일 CTA 배너 반복 = 해시 중복으로 저품질 직행
• 목업 합성: 매번 다른 배경과 합성으로 해시값 변경
• 미세 노이즈 주입: 1픽셀 색상 변경으로 해시 변조
• 재압축: JPEG 압축률 100%→99%로 바이너리 구조 변경
• 이미지 OCR: 전화번호 박힌 이미지 = 즉시 패널티
• CTA 비율: 목업합성 50%, 텍스트링크 30%, 고정배너 10% 미만

=== 무한 변주 프로토콜: 유사 문서 회피 3단계 시스템 ===
• 매번 글 생성 전 3가지 핵심 변수를 무작위 조합
• 경우의 수 수백 가지로 네이버 로봇 탐지 무력화

=== 🎭 화자 페르소나 로테이션 (어조 변형) ===
• Type A 현장 반장: 투박하고 거친 현장 용어, 감정적 경험 중심
• Type B 냉철한 행정가: 법률 규정 절차 중심, 건조한 신뢰감
• Type C 섬세한 상담사: 의뢰인 심리 공감 중심, 부드러운 해요체
• Type D 가상 의뢰인: 1인칭 피해자 시점, 감성 드라마
• Type E 업계 베테랑: "이 바닥에서 XX년" 식 경력 어필

=== 🏗️ 구조 파괴와 재조립 (플롯 변형) ===
• Pattern 1 두괄식 충격: After 사진부터 시작해 과거 역행
• Pattern 2 Q&A 인터뷰: 가상 의뢰인과 대화 형식
• Pattern 3 사건 일지: "09:00 현장 도착" 타임라인 보고서
• Pattern 4 실패와 극복: 문제 해결 중심 서사 구조
• Pattern 5 비교 분석: "A사 vs B사" 경쟁사 대조
• Pattern 6 사례 연구: "실제 의뢰 사례" 스토리텔링

=== 🎲 고유 변수 강제 주입 (디테일 변형) ===
• 현실의 불완전함을 강제로 심어 AI 감지 회피
• 날씨/계절: 장마철 습기, 한겨울 결로, 땡볕 더위
• 현장 특이사항: 엘베 없는 4층, 좁은 골목, 사나운 이웃
• 의뢰인 특징: 유학간 딸, 입원한 할머니, 몰래 이사
• 시간 특수성: 새벽 야반도주, 한낮 몰래, 주말 긴급
• 지역 특색: 구도심 골목, 신도시 아파트, 공단 원룸

=== 분량 증대 메커니즘 ===
• L2 논리적 확장: 배경 원인 심층 분석, 리스크 계층 분석
• L3 구조적 디테일: 4단계→8단계 세분화, 전문용어 해설
• 최소 분량: 2,000자 이상 목표 (자연스러운 정보 밀도)
• 매뉴얼 분할로 프로세스 상세화

=== YMYL 3단계 키워드 필터링 ===
• Tier 1 절대금지: 도박, 마약, 성인물, 불법대출
• Tier 2 YMYL: "치료", "완치", "특효", "100% 수익보장"
• Tier 3 과장광고: "최고", "1위", "국내 유일", "초특가"
• 문맥적 유해성: 단어 조합이 만드는 의미론적 독성 주의

=== IP 및 기술적 인프라 ===
• 고정 IP vs 유동 IP: 준최적 5-6단계는 고정 IP 권장
• 1계정 1IP 원칙, VPN은 통신사 고정 IP만
• 청정 IP 확보, IDC IP는 오염 위험
• 모바일 테더링 시 비행기모드로 IP 갱신

=== 올바른 상업적 콘텐츠 구조 ===
1. 서론: "우리는 누구인가" - 회사 정체성 확립
2. 서비스 영역: "무엇을 해결하는가" - 명확한 서비스 나열
3. 프로세스: "어떻게 해결하는가" - 투명한 절차 공개
4. 차별점: "왜 우리인가" - 기술력/법률자문/비밀보장
5. 행동유도: 스마트플레이스 지도 + 연락처

=== 스마트블록 대응 전략 ===
• 엔티티+속성 결합: [스타벅스] + [신메뉴 칼로리]
• 페르소나 타겟팅: "자취생을 위한", "직장인", "초보"
• 트렌드 블록: 정확한 날짜, 회차, 방송사 명시
• 메인 키워드 최전방 배치 (F-존 법칙)

=== 기존 공리들 ===

=== 1. 네이버 알고리즘의 이원화 구조 ===
• 일상 에세이 = D.I.A.+ 로직 (공감, 체류시간 중심)
• 상업 키워드 = C-Rank & 스팸필터 (출처 신뢰도, 상업성 감시)
• 흥신소 같은 고위험 키워드는 기본 점수가 깎인 상태에서 시작

=== 2. 체류시간 = 생존의 열쇠 ===
• 전화번호를 상단/중간에 두면 3초 이탈 → 지수 곤두박질
• 전화번호는 최하단 이미지나 지도에만 배치
• "읽을 수밖에 없는" 불안 해소 정보로 미끼 투척
• 예: "이걸 모르면 소송에서 집니다" → 30-60초 체류 강제 확보
• 3분 체류 시간이 상위노출 임계점

=== 3. 위장술 (Camouflage) = 상위노출의 핵심 ===
• "업체 홍보" 톤 금지 → "가상 의뢰인의 경험담"으로 위장
• "~~합니다, 연락주세요" (X) → "저도 이런 일을 겪었는데..." (O)
• 1인칭 감성 드라마 > 3인칭 전문가 리포트
• 화자를 탐정(나) → 가상의 피해자로 변경
• 공감하는 전문가 페르소나 = 진정성의 언어

=== 4. 키워드는 감정 뒤에 숨겨라 ===
• "#당진흥신소 #증거수집" (X) 
• "결국 당진흥신소 도움을 받기로 마음먹었어요" (O)
• 감정 서술 → 키워드 자연 삽입 → 스팸 필터 우회

=== 5. 이미지 OCR 함정 회피 ===
• 전화번호가 박힌 이미지 = 즉시 패널티
• 직접 촬영한 고유 이미지만 사용 (메타데이터로 진위 검증)
• 캡처 이미지 = 중복으로 처리되어 독창성 점수 0점

=== 6. 스마트블록 전략 ===
• 메인 키워드(흥신소) 진입장벽 높음
• 롱테일 키워드로 틈새 공략
• "배우자 외도 증거 수집 합법" 같은 구체적 의도 키워드

=== 7. 금칙어 및 위험 표현 ===
• 절대 금지: "최고", "유일", "보장", "100%", "완치"
• 의료/법률 분야 = YMYL 영역, 극도로 보수적 필터링
• "도청", "위치추적", "뒷조사" = 불법 서비스 연상 키워드

=== 8. 구조적 글쓰기 법칙 ===
• 제목의 키워드 = 본문 도입부에서 반드시 언급
• 키워드 스터핑 금지, 자연스러운 조사 활용으로 분산
• 기승전결 서사 구조 > 마케팅 템플릿
• 감정적 해소(카타르시스)로 완독률 확보

=== 9. D.I.A.+ 점수 최적화 ===
• 체류시간 > 클릭률
• 완독률 > 방문자 수  
• 자연스러운 댓글/공감 유도
• "Ghost User"(정보만 얻고 사라지는 유저) 방지
• 제목과 본문의 논리적 정합성 필수

=== 10. 최종 공리: "작가가 되어라" ===
• 기술적 SEO < 콘텐츠 기획(시나리오)
• 설득하는 글 < 유혹하는 글
• 전문가 리포트 < 감성 드라마
• 사실 전달 < 스토리텔링
• 단일 완결형 권위자 모델로 전환

=== 핵심 원칙 요약 - 8대 공리 체계 ===
1. 의도 일치가 모든 것의 시작점 (제1공리)
2. 반복 금지, 맥락과 자연스러움이 핵심 (제2공리)
3. 2분할 포스팅 절대 금지, 파워페이지만 (제3공리)
4. 제목 모바일 퍼스트 15자 승부 (제4공리)
5. 글 수정 절대 금지, 신뢰도 리셋 (제5공리)
6. 외부 유입 트랩 경계, 검색 유입 우선 (제6공리)
7. 텍스트 세탁 필수, 기술적 위생절차 (제7공리)
8. 이미지 해시 세탁, CTA 배너 관리 (제8공리)
9. 감정적 공감 → 전문성 어필 → 법적 안전성 강조 → 자연스러운 상담 유도
10. 위장된 후기 > 노골적 홍보
11. 체류시간 확보가 모든 지수의 시작점
12. 알고리즘을 속이되, 독자는 만족시켜라
